# NULL Functions Practice

In this practice session, I learned how NULL values work in SQL and why handling missing data is important in real-world databases. I understood that NULL values can affect calculations, filtering conditions and final query outputs if they are not handled properly.

I practiced different NULL handling functions and conditions used in employee, product and order-related datasets where some values like salary, bonus, discount, stock or category may be missing.

Topics Covered:
- IS NULL and IS NOT NULL to identify missing and available values
- IFNULL to replace NULL values with default values during calculations
- COALESCE to return the first available non-NULL value from multiple columns
- NULLIF to compare two values and return NULL when both are equal

I also practiced handling NULL values in:
- salary calculations
- bonus calculations
- discount handling
- product stock checking
- missing category identification

These queries helped me understand how SQL handles incomplete data and how NULL functions are useful in real-time reporting, data cleaning and business analysis scenarios.
